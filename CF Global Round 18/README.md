## e
一道数链剖分题，red尽量减少blue的最大操作数，blue尽量让操作数达到n/2。red尽量选取链长的叶子结点，因为一旦选取一个节点，它的祖先节点及其子🌲都无法被blue 选取。
应当尽量完整得讨论r/b的取值情况，当然也可以考虑使用O（n）复杂度的更新，因为n的范围并不算大。不要笑话这种思路，这种思路实际上更应该想到，因为讨论的时候其实很容易出错，简单好用才是我们考虑一个方法是否可行的标准，尤其是在比赛的时候。
这里的暴力更新其实也有说法的：
- 第一个循环：只有当k大于叶子结点的个数时才会进入，因此不会进入第二个循环，也就是blue没有办法给任何一个点上色
- 第二个循环：blue 上色数比较多，什么时候算比较多？就是b>r的时候（blue的潜在上色节点数大于red可以上色的数量（在red的最优博弈策略下）），这时blue需要考虑要不要把自己的额度用完；如果b < r 直接全部用完就好（得到的分数一定会尽量低）
总的来说，题目可以用暴力算法，毕竟很多时候是没有那么多时间来讨论情况-得出最终结论的，让计算用笨一点的方法得出结果有时是件好事。说白了还是一个取舍问题。
